# Cognitive Dividend Skill

そのまま使える AI スキルツールキット——『認知红利』の思考アップグレード方法論を、15 個の単独トリガー可能・組み合わせ呼び出し可能な認知ツールモジュールに分解しました。

これは本の要約ページではありません。本はこれらのスキルの源泉であり、このページの主役ではありません。

## このスキル群が解決する問題

- 忙しい感覚はあるのに成果が上がらず、注意力がどこに向いているかわからない
- 思考方式をアップグレードしたいが、どこから始めればいいか、どう練習すればいいかわからない
- 複雑な問題に対して直感に頼るだけで、構造化分析やシステム思考が欠けている
- 決断を下したいが、いつも二つの選択肢の間で迷い、第三の選択肢に気づかない
- 目の前の努力だけでなく、トレンドとポテンシャルエネルギーを理解したい

## 何が蒸留されたか

『認知红利』（謝春霖、2019）から 15 個の三重検証済みスキルを抽出し、5 つの次元をカバーしています：

- **富の再理解**：`attention-management`（注意力管理）、`time-merchant`（時間商人四モード）、`compound-effect`（複利効果設計）
- **自分の再理解**：`nlp-levels`（NLP 六段階モデル）、`metacognition-control`（メタ認知三段階）、`multi-dimension-ability`（多次元能力構築）
- **世界の再理解**：`potential-energy`（ポテンシャルエネルギー差と点線面体）、`mirror-world`（鏡の世界と四領域空間）
- **思考力向上**：`brain-unseal`（脳の封印解除）、`dialysis-prism`（透析プリズム）、`structured-thinking`（構造化思考と MECE）、`systems-thinking`（システム思考）
- **問題解決**：`scientific-decision`（科学的意思決定四ステップ法）、`evolution-strategy`（進化戦略）、`innovation-method`（イノベーション方法論）

## 効果例

### 例 1：複雑な決断に直面した時

**ユーザーの質問**

> 「二つの求人のオファーで迷っています。一つは給料が高いが不安定、もう一つは安定但し天井が低い。どう選べばいい？」

**スキルの判断**

- `nlp-levels` をトリガーし、決断を「行動レベル」から「アイデンティティレベル」と「ビジョンレベル」に引き上げる
- `scientific-decision` を呼び出し、四ステップ法で選択問題を計算問題に変換
- 長期的なパスが関係する場合、`compound-effect` と `potential-energy` を補足

**回答例**

> 二つの選択肢から一つ選ぶのではなく、まず一段上に跳びましょう：あなたはどんな人になりたいですか？この問いの答えが、どちらの選択肢がそもそも迷う価値もないかを直接決定します。その後、科学的意思決定の四ステップ法を適用します：まず目的を明確にする（「どちらの仕事を選ぶか」ではなく「5 年後どこにいたいか」）、全代替案を列挙する（この二つのオファーだけではない）、条件で絞り込む、最後にスコアリングして比較する。ほとんどの迷いは、二つの選択肢が同等だからではなく、間違ったレベルで選択しているからです。

### 例 2：チームは忙しいが成果が出ない

**ユーザーの質問**

> 「チームは毎日忙しくしていますが、重要なプロジェクトが全く進みません。何が問題でしょうか？」

**スキルの判断**

- `attention-management` をトリガーし、注意力のブラックホールをチェック
- `systems-thinking` を呼び出し、個人の問題ではなくシステム構造の問題かどうかを確認
- `structured-thinking` を補足し、混乱した情報を整理

**回答例**

> まず問題がどのレベルにあるかを判断します：全員が忙しいのに成果が基準に達しない場合、おそらく「努力が足りない」問題ではなく、注意力の配分に問題があります。最も一般的な三つのブラックホールは：受動的対応（メッセージ駆動の仕事）、断片切替（頻繁なタスク切り替え）、非効率な繰り返し（自動化すべきことを手動でやっている）。次にシステム構造を見ます：インセンティブは誤った行動を促していないか？フィードバックループは強化型かバランス型か？「忙しいが無効」の本質は、多くの場合、システム構造が注意力を低テコの位置に吸い寄せていることです。

## これらのスキルはどう生成されたか

これらのスキルは **cangjie-skill** の RIA-TV++ パイプラインを使用して生成されました。全行程は六つの段階に分かれています：

1. **全書理解（Adler 分析）**——Mortimer Adler の分析的読書法を用い、全書を構造・解釈・批判・応用の四ステップで分解。`BOOK_OVERVIEW.md` を产出
2. **並列抽出**——5 つの専門抽出器（フレームワーク、原則、事例、反例、用語）を同時に配備し、原文から候補方法論ユニットを抽出
3. **三重検証スクリーニング**——各候補は三つの検査に合格する必要がある：書中に少なくとも 2 箇所の独立した裏付けがある（クロスドメイン）、書中で明示されていない新しい問いに答えられる（予測力）、常識ではない（独自性）
4. **RIA++ 構築**——検証済みの内容を六つの次元で構造化：R（原文引用）/ I（自分の言葉で書き直し）/ A1（書中の事例）/ A2（将来のトリガーシナリオ）/ E（実行可能ステップ）/ B（境界と盲点）
5. **Zettelkasten リンク**——スキル間の依存・対比・組み合わせ関係を見出し、`INDEX.md` と参照グラフを生成
6. **ストレステスト**——各スキルに囮問題を含むテストケースを設計。不合格のスキルは作り直し

## 使い方

1. `INDEX.md` を閲覧してスキルの全体像と依存関係を把握
2. 現在の問題に関連する `*/SKILL.md` を見つけ、トリガー条件と実行ステップをそのまま使用
3. スキルをエージェントフレームワークに統合するか、プロンプトを独立したツールとして使用
4. `test-prompts.json` を使って、正しいシナリオでトリガーされること・誤ったシナリオでトリガーされないことを検証
